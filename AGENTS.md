# AGENTS.md

## Role For Codex

Codex works on this repository as a focused Unity implementation assistant for the NavMesh diploma demo.

Keep the work small and demonstrable:

- make targeted changes instead of rewriting the project;
- do not add complex features unless explicitly requested;
- keep the purpose of the demo visible: NavMesh navigation, route costs, dynamic obstacles, and multiple NPC agents;
- do not touch `Library`, `Temp`, `Logs`, or `obj`;
- normally work only in `Assets`, `Packages`, `ProjectSettings`, and markdown documentation;
- after changes, briefly report what changed, which files were touched, and how to check the result in Unity.

## Suggested Roles

These are lightweight roles for future tasks. They are documentation roles, not extra tooling.

- `Demo Reviewer`: checks whether the scene clearly proves the intended navigation idea.
- `Unity Implementer`: makes small Unity/script changes and keeps the demo stable.
- `Documentation Maintainer`: updates README, checklist, and teacher-facing notes after changes.
